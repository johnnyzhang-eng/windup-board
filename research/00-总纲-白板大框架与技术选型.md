## 一、白板讲的大框架（结构化）

一条主管线，穿插分类维度和待验证实验：

```
① 图片(正视) → 侧面图        分类：伪3D / 2D(低像素/高像素)
                            分叉：视频截图也是一种得侧面的路子
                            图片大小随游戏类型/限制变(→256×256画布)
        ↓
② 动画                       帧数：4/6/8/16/32(实验变量)
   ├ 动作：预设(待机/跑/跳/攻击) + 自定义(漂浮/瘸腿…)
   ├ 转身 → 4向/8向图
   ├ 一致性(核心)：人物/肢体/衣物/手持物 一致
   │    手段：回退(上一帧约束) · 伪3D→骨骼 · 低像素2D→像素/高像素→结合?
   └ 生成不一致(核心痛点)：提示词过复杂 · 人物瑕疵(能否不生图修正?)
        · 识别生成不一致 → 评判标准(数两条腿、查脸部)
        ↓
③ 抠图(工具AI)               缺陷 · 背景颜色智能选择 · 像素抠图

右侧流程(已落地一条)：人设图→生成→精灵图(4/6/8帧,idle sheet)→切分去背→单张256×256
                    →对齐→函数映射(比例缩放)→导入Cocos。
                    另一条：母版生成/替换(模型选择需A/B Test、Prompt)
                    待验证标注：1.约束性不强? 2.非人?
```

## 二、六个方向的技术选型结论（一句话概览）

| 方向 | 现在就用 | P1（下一步） | 不用/慎用 |
|---|---|---|---|
| **跨帧一致性** | 分层配方：**角色 LoRA(身份) + ControlNet-OpenPose(姿势)**；已有母版+VLM门禁 | IP-Adapter FaceID(数据不足兜底)、Sprite Sheet Diffusion | 固定seed当一致性手段(反效果)、reference-only(太弱) |
| **坏帧不生图直接修** | **mask-composite 局部修**（LaMa擦除 / Flux Fill补道具 / 低denoise微修） | Differential Diffusion去接缝、Gemini/Kontext当语义fallback | 整图重生成当默认(会全局漂移破坏对齐) |
| **风格分路线** | 像素线必接**强制网格化后处理**(unfake.js)；高清线用IP-Adapter+ControlNet+BiRefNet抠图 | Retro Diffusion/PixelLab对标 | 像素线用AI matting(产半透明边) |
| **骨架/3D驱动动作** | walk预处理器换**DWPose**；**Mixamo→OpenPose**当动作库；3D只出控制图(保单帧可编辑) | side-on叠Depth补OpenPose侧面弱点、toyxyz多通道 | Sprite Sheet Diffusion(无开源码) |
| **多方向/转身** | 先稳做**4向**、对称角色镜像省一半 | 单次grid turnaround、MV-Adapter、3D渲相机 | 单次直出8向(当前模型做不稳) |
| **自动评判(数腿/脸)** | **DreamSim**测漂移、**VLM-as-Judge**(rubric)查语义、**CLIP-IQA**查脸崩 | 关键点计数查多腿、warping error查闪烁 | ❌**别让VLM"数腿"**(准确率仅2%,靠先验背答案) |

## 三、最关键的几条设计原则（从研究里提炼）

1. **一致性没有单一银弹，是分层组合**：LoRA锁身份 + OpenPose锁姿势 + (兜底)IP-Adapter锁外观。我们缺的是"把身份烧进LoRA"这一层——已有母版正好当训练集。
2. **坏帧修复默认走 mask-composite，不整图重生成**：只改坏的那一块、其余像素逐像素不动 → 天然保住帧间对齐和单帧可编辑。这直接回答白板"能否不生图修正"=能，且是首选。
3. **3D 只出"控制图"(骨架/深度)、不出成品**：每帧仍逐帧独立生成 → 保住单帧可编辑；一致性用seed+参考锁，不用视频模型。这是"伪3D→骨骼"的正确姿势。
4. **像素画必须生成后接强制网格化**（scale检测→网格吸附→调色板量化）；纯靠prompt/模型拿不到硬网格。
5. **红线：计数类语义(腿/手指/道具数)绝不问VLM**——它靠先验背答案、准确率2%。计数走几何/关键点。
6. **4向先达标、8向P2**：单次直出8向当前会漂成"8个变体"而非"同一角色8朝向"。

## 四、分风格的推荐主链路

**像素线**：pixel-art-xl LoRA(固风格) + 角色LoRA(叠身份) → OpenPose逐帧 → 键色抠图+alpha二值化 → unfake.js强制网格化 → 对齐 → 引擎。
**高清线**：角色LoRA/IP-Adapter FaceID(锁身份) → ControlNet-OpenPose/DWPose(锁姿势) → BiRefNet/RMBG抠图 → Differential Diffusion修接缝 → 对齐 → 引擎。
**伪3D线**：Zero123++/SV3D 或 Blender建模 → 多角度渲染 → (可选像素化) → 抠图 → 引擎。一致性天花板但成本高。
