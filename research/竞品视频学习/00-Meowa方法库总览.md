## 二、方法库（按我们 pipeline 关心点归类，术语已校正）

### A. 动画生成（HIGH 那期，最值钱）
- **两条路线**：高清=视频模型（清晰度高）；像素=像素模型或关键帧。生成后 → 拆序列帧 → 选帧 → 一键去背景 → 存动图。
- **★ 参考图=动画首帧；用"运动中"的角色图当首帧、别用站立图**（或直接删掉首帧）→ 避免"站立→运动"硬过渡、**循环更顺**。（直接解我们纠结的循环接缝问题，比抽 steady-cycle 更根本）
- **★ 颜色数 24/48**：太少丢细节、太多帧间细微色变→**闪烁/噪点**（= 我们说的像素 boiling；他们用"固定色数"这个旋钮压它）。
- **padding 画布留白 + 九宫格构图**：给动作（尤其攻击）留空间，塞太满动作展不开——最常见坑。
- **关键帧 + 强度**：补全中间动作，可控（对应我们"控制动作/单帧"）。
- **帧数默认 8 可调**：少不流畅、多出多余动作（印证我们 [[08-帧数与风格分辨率匹配]]）。

### B. 完美像素（强制网格化）
- 像素模板输出即"完美像素"；非像素图/大图先走"像素化"→完美像素（= 我们 unfake 验证过的强制网格化）。
- **模型选型信号：像素首选 Nano Banana Pro（更遵循像素逻辑、少扭曲），GPT Image2 像素实用性差**——对 E2 模型选型 / E4a 有用。

### C. 进引擎最后一公里（他们做全了，我们的护城河也在这）
一键去背景、序列帧抽取、动图/序列帧导出、结构化地图数据导出、地块自动吸附对齐、无缝循环平铺。

### D. 多方向 / 转身（对应 E7 #15）
- 八向图：单图一键出 8 向、**自动去背景 + 按方向拆分**；简单动作(待机)可整八向一次生成。
- 模板可设**"默认视角"**字段。

### E. 单帧可重画（我们的卖点，他们有产品范例）
- 局部改图：**保尺寸、保完美像素、非改动区域与原图像素级一致**，可直接进引擎——正是"改一处不动其他"的现成范例。

### F. 模板驱动稳风格（对应 E3/E4）
- 自定义模板：角色模板(正好2张参考) / 素材包模板(3-8张)；参考图须规范像素图 + 去背景；**参考图尺寸=模板尺寸=在模板层锁分辨率**；"风格对齐"开关贴近参考。

### G. 工作流 / agent 集成（他们真正的护城河）
- 网页 / Skill / Agent 三形态。Skill 接进 Codex 类工具，**agent 读项目上下文（像素规格/帧数/接入方式）自动生成并替换素材**。

## 三、对我们的行动项（可落地）
1. **动画首帧用"运动中"帧**（A）→ 试进我们 route-B / 逐帧：应能显著改善循环接缝，比抽 steady-cycle 更省。→ 记入动画实验。
2. **像素 boiling 用固定颜色数(24/48)压**（A/B）→ 补进 E4a #10（我们之前靠共享调色板，他们点明"色数档位"是关键旋钮）。
3. **padding+九宫格构图**（A）→ 作为像素动画生成的质量旋钮，记入 E4a/E6。
4. **像素模型首选 Nano Banana Pro > GPT Image2**（B）→ 补进 E2 模型选型台 #8。
5. **模板"默认视角"字段 + 八向自动拆分**（D）→ 补进 E7 #15。
6. **守护城河**：他们把"进引擎最后一公里 + workflow/agent 集成"做全了；我们的差异化（可量化一致性 DreamSim/E1、真·单帧可重画、国内适配）必须**做成 skill 暴露在开发者干活的地方**，否则再好没人用。
