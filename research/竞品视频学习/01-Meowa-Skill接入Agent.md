# 竞品视频学习 · Meowa 频道 · 01

## 【Meowa教程·Skill版】AI读懂游戏，自己生成素材（第二期）
- 视频：BV1X8G26HEyR（约 1 分半）
- 提取法：yt-dlp 下音频 → faster-whisper base（模型走魔搭 ModelScope，本机 ASR，¥0）→ 我校正术语综合
- 主题：把 Meowa 作为 **Skill 接入 AI coding agent（Codex/CodexCloud）**，让智能体在项目里直接生成并替换游戏素材

### 工作流（已校正 base 模型术语：声称=生成/Skyo=Skill/向速=像素/真数=帧数）
1. **安装**：把 Skill 链接发给 agent → 它自动安装（可选仅当前项目 / 全局）。
2. **接 API**：Meowa 网站建 API key → 放进项目 `.env` 私密文件（勿泄露）；实在不行让 agent 去配。
3. **用**：自然语言下指令（例："把游戏怪物掉落的金币换成多种冰淇淋"）→ agent **判断项目风格**后生成。
4. **不满意**（如像素尺寸太大）→ 让 agent 列出合适模板 → 选**像素尺寸 + 主题都接近**的道具模板重生成。
5. **质量过关** → 让 agent **直接替换**项目里的素材。
6. **动画**：挑一张原图 + 简单描述动作 → Skill **自动确定尺寸和帧数**（不合适可手动）→ 直接替换游戏敌人。
7. **音效/BGM**：音效可定时长、一次生成多个挑选；BGM 让 agent **自动截取游戏画面/时机作参考** → 更贴氛围。

### 他们亲口强调的关键优势
相比单独用网页端，Skill/agent **主动读取项目的真实需求**——**像素规格、动画帧数、接入方式**都能自动读取对齐，不用手填。

### 对我们（Windup）的启示
1. **他们真正的护城河是"workflow 集成 + 项目上下文自动读取"，不只是生成质量**。素材在 agent/IDE 里直接生成+替换、自动读项目的像素规格/帧数/接入方式。→ 我们的"一致性可量化 / 单帧可重画 / 进引擎"再好，也**得暴露在开发者干活的地方（做成 skill）**，否则没人用。
2. **它自动读"动画帧数"** 正好印证我们 [[08-帧数与风格分辨率匹配]]——他们把"帧数按分辨率定"做进了 skill 的自动判断。
3. **模板驱动**再次出现（尺寸+主题接近的模板重生成）——一致性靠模板锁，对上我们 E3/E4。
4. 分发模式 = skill-as-loader（见 [[09-竞品Holonic-Asset-角色设计对比]] 里提到的自更新 skill），这期是实操演示。

### 方法可行性备注
- **无字幕**（只有弹幕）→ 必须 ASR。base 模型术语错听多，**批量建议换 medium**（中文更准）或我逐条校正。
