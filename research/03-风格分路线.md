# 03 · 不同美术风格走不同路线（做成巨大 pipeline 的分叉依据）

> 白板：低像素2D / 高像素 / 伪3D 走不同路线。核心结论：**像素画必须"生成→强制网格化后处理"两段式**，纯 prompt/模型拿不到硬网格。

## 为什么像素画最难
扩散模型天然产的正是像素画的反面——像素大小不一、描边不连续、模糊、抗锯齿噪点、幻觉出不对齐的网格。这是机制根本矛盾，专门训练的像素模型也有。→ **像素线必须在生成后接一道强制网格化，不能省。**

## 三条风格线的主链路

### 像素线
| 环节 | 用什么 |
|---|---|
| 风格+身份 | pixel-art-xl LoRA(权重1.2, 配 LCM 8步/guidance1.5) + 角色 LoRA 叠加 |
| 抠图 | **键色/角落色 + alpha 二值化**（❌ 禁用 AI matting，会产半透明边破坏硬边） |
| 后处理(必接) | **unfake.js**：scale检测→网格吸附→调色板量化→形态学去噪→alpha二值化。或 PixelRefiner |
| 商业对标 | Retro Diffusion(自研 pixel decoder,解码即硬网格)、PixelLab(4/8向+skeleton动画,有API+MCP) |
| 前沿 | SD-πXL(硬palette约束生成, P2 research级) |

### 高清手绘线
| 环节 | 用什么 |
|---|---|
| 一致性 | IP-Adapter FaceID Plus v2 锁脸 + ControlNet(OpenPose)锁姿 + ADetailer精修脸 → 80-95%；冲近100%上自训LoRA |
| 改朝向/动作 | **指令编辑模型**（Flux Kontext / Qwen-Image-Edit / Nano Banana）比重新txt2img一致性高一个量级。Nano Banana 10轮编辑后仍>96%一致 |
| 抠图 | **BiRefNet / RMBG-2.0**（连续alpha,头发/半透明边平滑）；用 ComfyUI-RMBG 一键横评选型 |
| 多视角 | turnaround LoRA（一帧出前/侧/背） |

### 伪3D / 2.5D 线
| 路线 | 方法 | 结论 |
|---|---|---|
| AI 单图→多视角 | Zero123++ / Stable Zero123（6视角3×2 tiling） | P1，参考图→侧面最短路 |
| 3D建模→渲染（一致性天花板） | Blender(BlenderSpriteGenerator/Foozle/Sorceress)绕角色布相机渲各角度，正交相机 | P1，一致性物理级100%，成本=一次建模 |
| 单图→3D角色 | CharacterGen（比通用 Zero123 更懂人体） | P2 |
