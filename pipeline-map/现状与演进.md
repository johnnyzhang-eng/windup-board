# Windup 大 Pipeline · 现状与演进地图

> 这是"巨大 pipeline 不停完善"的全景。左=已实现，右=研究确认要接入的。持续更新。

## 已实现（huyanxius/windup-asset-lab 后端 + 本组 pipeline 内核）

```
前端(asset-lab)                后端(server/windup_pipeline)
创建角色页 → POST /api/characters/generations → 后台线程 GenerationExecutor
                                                    ↓
① gen_character 母版生成(伪侧面朝右, 品红底)
② matte_chroma 键色抠图 (前景>60%回退AI抠, 两次失败拒绝)
③ 母版门禁: VLM 判 view/facing, 不是朝右侧面→重生成→仍不合格整包失败
④ 每动作 8 帧, 两套机制:
   · idle → gen_action_sheet 一次出8格横条 → 切分 → 全条共用scale对齐
   · walk → 强制骨架引导逐帧: skeleton_gen算8帧关节角 → gen_frame(母版+上一帧+骨架)×8
⑤ normalize_frame 脚底基线贴 256×256
⑥ sequence_quality 几何质检(高度/中心/脚底/覆盖率) + semanticReviewRequired 恒真转人工
⑦ job-poller 900ms 轮询, 帧逐张点亮
```

## 研究确认要接入的（按优先级）

**P0（现在就该接，研究给了明确方案）**
- 一致性补 **角色 LoRA 层**（母版当训练集，身份烧进权重）——现在只靠母版参考，漂移压不到最低
- 坏帧修复接 **mask-composite**（LaMa擦除 / Flux Fill补道具 / 低denoise微修）——白板"能否不生图修正"落地
- 像素线接 **unfake.js 强制网格化后处理**——像素保真必需
- walk 预处理器换 **DWPose**、动作库接 **Mixamo→OpenPose**
- 评判补 **DreamSim(漂移) + VLM-as-Judge(缺道具/朝向) + 关键点计数(数腿)**
- 红线：**计数不问 VLM**

**P1（下一步）**
- 4向 turnaround（grid 或 MV-Adapter）；side-on 叠 Depth；高清抠图换 BiRefNet/RMBG；Sprite Sheet Diffusion 评估；warping error 时序门禁

**P2（远期）**
- 8向；伪3D(Zero123++/SV3D/Blender渲相机)；top-down/isometric；SD-πXL 硬palette
